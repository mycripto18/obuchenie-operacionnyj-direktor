# Инструкция по миграции на статическую архитектуру

## Обзор новой архитектуры

- **Публичный сайт**: чистый HTML + CSS (без React, без JavaScript)
- **Админка**: React-приложение только по адресу `/panel-x7k9m2`
- **Деплой**: статический `index.html` + `styles-static.css`

---

## Часть 1: Экспорт контента со старого сайта

### Шаг 1.1: Скачивание content.json

**Вариант А — через админку (если есть):**
1. Открыть старый сайт: `https://старый-сайт.vercel.app/panel-x7k9m2`
2. Ввести пароль (обычно `admin123`)
3. Нажать кнопку **"Скачать JSON"** или **"Экспорт"**
4. Сохранить файл `content.json` на компьютер

**Вариант Б — напрямую из GitHub:**
1. Открыть репозиторий старого проекта на GitHub
2. Перейти в папку `public/`
3. Найти файл `content.json`
4. Нажать кнопку **"Raw"** → сохранить страницу как `content.json`

**Вариант В — через URL:**
1. Открыть в браузере: `https://старый-сайт.vercel.app/content.json`
2. Сохранить страницу как `content.json` (Ctrl+S / Cmd+S)

### Шаг 1.2: Скачивание изображений

**Найти изображения в старом проекте:**

```
Где искать:
├── public/images/          ← Локальные изображения
├── public/uploads/         ← Загруженные файлы
├── src/assets/             ← Иногда здесь
└── content.json            ← Ссылки на внешние URL
```

**Если изображения локальные:**
1. Открыть репозиторий на GitHub
2. Перейти в `public/images/` (или `public/uploads/`)
3. Скачать каждый файл:
   - Нажать на файл → **"Download"** или **"Raw"** → сохранить

**Если изображения внешние (URL):**
1. Открыть `content.json` в текстовом редакторе
2. Найти поля: `schoolLogo`, `photo`, `image`
3. Открыть каждый URL в браузере → сохранить изображение
4. Записать соответствие: какой файл для какого курса/автора

### Шаг 1.3: Создание резервной копии

```bash

# - любые кастомные стили (если были)
```

**Структура бэкапа:**

## Часть 2: Удаление старых файлов

### Шаг 2.1: Клонирование репозитория

```bash
# Клонировать старый проект (если ещё не склонирован)
git clone https://github.com/ваш-аккаунт/старый-проект.git
cd старый-проект
```

### Шаг 2.2: Сохранение .git папки

**ВАЖНО: НЕ УДАЛЯЙТЕ папку `.git`!**

```bash
# Проверить, что .git существует
ls -la | grep .git

# Должно показать:
# drwxr-xr-x  .git
```

### Шаг 2.3: Полное удаление старых файлов

**Вариант А — через терминал (рекомендуется):**

```bash
# Удалить ВСЁ кроме .git
find . -maxdepth 1 ! -name '.git' ! -name '.' -exec rm -rf {} +

# Проверить — должна остаться только .git
ls -la
# Показывает только: . .. .git
```

**Вариант Б — через файловый менеджер:**

1. Открыть папку проекта
2. Включить отображение скрытых файлов:
   - **Windows**: Вид → Скрытые элементы
   - **Mac**: Cmd + Shift + .
   - **Linux**: Ctrl + H
3. Выделить ВСЕ файлы и папки **КРОМЕ** `.git`
4. Удалить (в корзину для безопасности)

**Что должно остаться:**
```
старый-проект/
└── .git/                ← ТОЛЬКО ЭТА ПАПКА!
```

### Шаг 2.4: Проверка

```bash
# Git должен показать, что все файлы удалены
git status

# Вывод будет примерно таким:
# deleted:    README.md
# deleted:    package.json
# deleted:    src/App.tsx
# ... (много удалённых файлов)
```

## Часть 4: Импорт контента и изображений

### Шаг 4.1: Загрузка изображений

1. **Создать папку для изображений:**
```bash
mkdir -p public/images
```

2. **Скопировать изображения из бэкапа:**
```bash
cp backup-старый-сайт/images/* public/images/
```

3. **Проверить структуру:**
```
public/images/
├── logo-skillbox.png
├── logo-netology.png
├── author-photo.jpg
└── ...
```

### Шаг 4.2: Обновление путей в content.json

**Если изображения были внешними URL — оставить как есть.**

**Если изображения стали локальными — обновить пути:**

Открыть `backup-старый-сайт/content.json` и заменить:

```json
// БЫЛО (внешний URL):
"schoolLogo": "https://example.com/images/skillbox.png"

// СТАЛО (локальный путь):
"schoolLogo": "/images/logo-skillbox.png"
```

```json
// БЫЛО:
"photo": "https://external-site.com/author.jpg"

// СТАЛО:
"photo": "/images/author-photo.jpg"
```

### Шаг 4.3: Запуск проекта локально

```bash
cd старый-проект

# Установить зависимости
npm install
# или
bun install

# Запустить
npm run dev
# или
bun dev
```

Открыть: `http://localhost:5173/panel-x7k9m2`

### Шаг 4.4: Импорт контента через админку

1. **Открыть админку:** `http://localhost:5173/panel-x7k9m2`
2. **Ввести пароль:** `admin123`
3. **Импортировать JSON:**
   - Перетащить файл `content.json` в зону загрузки
   - **ИЛИ** нажать "Выбрать файл" и выбрать JSON
4. **Выбрать страницу:**
   - В диалоге выбрать **"Главная страница"**
   - Нажать **"Импортировать"**
5. **Сохранить:**
   - Нажать кнопку **"Сохранить"**
   - Дождаться статуса "Сохранено"

### Шаг 4.5: Проверка изображений

1. Перейти во вкладку **"Курсы"**
2. Проверить, что логотипы школ отображаются
3. Если не отображаются:
   - Проверить путь (должен начинаться с `/images/`)
   - Проверить, что файл существует в `public/images/`
   - Проверить регистр букв (Linux чувствителен к регистру)

### Шаг 4.6: Исправление путей изображений (если нужно)

**В админке:**
1. Открыть вкладку "Курсы"
2. Для каждого курса:
   - Найти поле "Логотип школы"
   - Ввести правильный путь: `/images/название-файла.png`
3. Открыть вкладку "Автор"
   - Проверить поле "Фото автора"
4. Нажать **"Сохранить"**

## Решение проблем

### Изображения не загружаются

**Проверить:**
1. Путь начинается с `/images/` (не `images/` без слеша)
2. Файл существует в `public/images/`
3. Регистр букв совпадает (Linux чувствителен)
4. Формат файла поддерживается (jpg, png, webp, svg)

**Исправить в content.json:**
```json
"schoolLogo": "/images/school-logo.png"  ✓
"schoolLogo": "images/school-logo.png"   ✗
"schoolLogo": "./images/school-logo.png" ✗
```

### Стили не применяются

1. Проверить `<link>` в index.html:
```html
<link rel="stylesheet" href="/styles-static.css">
```
2. Проверить, что `public/styles-static.css` существует
3. Скачать свежий CSS из вкладки "Статика"

### Пустая страница

1. Открыть DevTools (F12) → Console
2. Проверить ошибки
3. View Source — должен быть полный HTML, не React-заглушка

### Забыли пароль админки

1. DevTools (F12) → Application → Local Storage
2. Удалить ключ `admin_password`
3. Обновить страницу
4. Пароль сброшен на `admin123`
