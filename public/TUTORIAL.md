## 1. Главная страница сайта

Главная страница содержит:

- **Шапка** — заголовок статьи с количеством отзывов и датой обновления
- **Навигация** — быстрые ссылки на разделы сайта
- **Блок автора** — информация об авторе статьи с фото
- **Вступительный текст** — краткое описание темы
- **Критерии выбора** — на что обращать внимание при выборе курса
- **Список курсов** — карточки с кратким описанием курсов
- **Детали курсов** — подробная информация о каждом курсе
- **Контентные блоки** — дополнительная информация
- **FAQ** — часто задаваемые вопросы
- **Рекламная пометка** — дисклеймер о рекламном характере
- **Футер** — ссылки на правовые страницы и контакты

## 6. Редактирование контента

### Вкладка "Контент"

**Мета-данные** (SEO):
- `Title` — заголовок страницы (до 60 символов)
- `Description` — описание (до 160 символов)
- `Keywords` — ключевые слова через запятую
- `Canonical URL` — каноническая ссылка

**Статистика в шапке**:
- Количество отзывов
- Текст бейджа
- Подзаголовок

**Вступительный текст**:
- Редактор с поддержкой форматирования

### Вкладка "Автор"

- **Имя** — ФИО автора
- **Фото** — URL изображения или загрузка
- **Описание** — краткая биография
- **Ссылка** — URL профиля

## 7. Управление курсами

### Добавление курса

1. Перейдите на вкладку "Курсы"
2. Нажмите кнопку "Добавить курс"
3. Заполните все поля

### Поля курса

| Поле | Описание |
|------|----------|
| Название | Название курса |
| Школа | Название онлайн-школы |
| Логотип школы | URL логотипа |
| Ссылка | URL курса (партнёрская ссылка) |
| Цена | Текущая цена |
| Старая цена | Цена до скидки |
| Рассрочка | Ежемесячный платёж |
| Формат | Онлайн/Офлайн/Смешанный |
| Длительность | Срок обучения |
| Документ | Сертификат/Диплом |
| Для кого | Целевая аудитория |
| Бейдж | top/popular/new |

### Промокод

- **Код** — текст промокода
- **Описание скидки** — "Скидка 10%"
- **Процент** — числовое значение

### Сортировка курсов

- Перетаскивайте курсы мышкой для изменения порядка
- Используйте кнопки ↑↓ на мобильных

### Массовая замена ссылок

1. Нажмите "Массовая замена URL"
2. Вставьте ссылки (по одной на строку)
3. Порядок соответствует порядку курсов

### Массовая замена промокодов

Формат: `ПРОМОКОД` или `ПРОМОКОД|Текст скидки|Процент`

```
SALE2024
PROMO|Скидка 15%|15
```

## 8. Управление страницами

### Создание страницы

1. Вкладка "Страницы" → "Добавить страницу"
2. Заполните slug (URL), заголовок, описание
3. Включите/выключите нужные блоки

### Настройки страницы

- **Slug** — часть URL (например: `java-courses`)
- **Заголовок** — для меню и мета-тегов
- **Показывать в меню** — вкл/выкл
- **Блоки** — какие секции отображать

### Контент страницы

Каждая страница может иметь:
- Свой заголовок
- Своего автора
- Свои курсы
- Свой FAQ
- Свои контентные блоки

### Дублирование страницы

Нажмите кнопку "Дублировать" для создания копии со всем контентом.

## 9. Правовые страницы

### Управление правовыми страницами

1. Вкладка "Правовые" в админке
2. По умолчанию создана "Политика конфиденциальности"
3. Можно добавлять любые страницы (Условия использования, Оферта и т.д.)

### Настройки правовой страницы

- **Заголовок** — название страницы
- **Slug** — URL (например: `terms` → `/legal/terms`)
- **Показывать в футере** — добавит ссылку в подвал сайта
- **Разделы** — заголовок + текст (поддерживаются списки через `•` или `-`)

### Форматирование текста

```
Основной текст параграфа.

• Пункт списка 1
• Пункт списка 2
- Также работает с дефисом

Следующий параграф.
```

## 10. Экспорт и импорт данных

### Объединённый экспорт/публикация

1. Нажмите кнопку **"Экспорт / Публикация"** в шапке админки
2. Выберите что скачать:
   - 📦 **Весь сайт** — для публикации (полный `content.json`)
   - 📄 **Главная страница** — для резерва/переноса
   - 📄 **Дополнительная страница** — для резерва одной страницы
3. Нажмите **"Скачать"**

### Импорт из JSON

1. Перетащите файл в зону загрузки или нажмите "Выбрать файл"
2. В диалоге выберите куда импортировать:
   - 📦 **Весь сайт** — заменит весь контент
   - 📄 **Главная страница** — заменит только главную
   - 📄 **Конкретная страница** — заменит только выбранную
3. Нажмите **"Импортировать"**
4. **Обязательно нажмите "Сохранить"!**

### Типы файлов

| Тип | Назначение | Действие после скачивания |
|-----|------------|---------------------------|
| 📦 Весь сайт | Публикация | Заменить `public/content.json` → git push |
| 📄 Страница | Резерв/Перенос | Импортировать обратно в ту же страницу |

## 11. Публикация на хостинг

### 🆕 Рекомендуемый способ: Статический HTML (без JavaScript)

Теперь вы можете создать **полностью статический сайт** без React и JavaScript:

1. Откройте админку: `/panel-x7k9m2`
2. Перейдите на вкладку **"Статика"** (🌐)
3. Нажмите **"Скачать index.html"** для главной страницы
4. Замените файл `public/index.html` скачанным файлом
5. Для доп. страниц создайте папки `public/[slug]/` и положите туда index.html
6. Убедитесь, что `public/styles-static.css` присутствует
7. Выполните git push — Vercel задеплоит статический сайт

**Преимущества статического HTML:**
- ✅ Полная индексация Яндексом (без JavaScript)
- ✅ Мгновенная загрузка страниц
- ✅ Работает без сервера и без React
- ✅ Идеально для SEO

**Структура папок для статического сайта:**
```
public/
├── index.html          ← Главная (статическая)
├── styles-static.css   ← Стили
├── favicon.png
├── kursy-java/
│   └── index.html      ← Доп. страница
├── kursy-python/
│   └── index.html      ← Доп. страница
└── legal/
    └── privacy/
        └── index.html  ← Правовая страница
```

### Альтернативный способ: React SPA + JSON

Если нужна динамика (React):

1. В диалоге "Экспорт / Публикация" выберите **"Весь сайт"**
2. Файл `content.json` скачается автоматически
3. Замените `public/content.json` в репозитории
4. Выполните команды:
   ```bash
   git add public/content.json
   git commit -m "Update content"
   git push
   ```
5. Vercel автоматически обновит сайт

### Ручная сборка для других хостингов

1. Склонируйте репозиторий
2. Замените `public/content.json`
3. Соберите проект:
   ```bash
   npm install
   npm run build
   ```
4. Загрузите содержимое `dist/` на хостинг
5. Для Apache создайте `.htaccess`:
   ```apache
   RewriteEngine On
   RewriteBase /
   RewriteRule ^index\.html$ - [L]
   RewriteCond %{REQUEST_FILENAME} !-f
   RewriteCond %{REQUEST_FILENAME} !-d
   RewriteRule . /index.html [L]
   ```

## Синхронизация между устройствами

### Как это работает

1. Все устройства (ПК, мобильные, инкогнито) загружают контент из `public/content.json`
2. Админка редактирует локальную копию в браузере (localStorage)
3. После публикации (замены `content.json` в репо) все устройства увидят обновления

### Важно

- Изменения в админке видны **только в текущем браузере** до публикации
- Для синхронизации между устройствами **обязательна публикация** через GitHub
- После git push обновления появятся на всех устройствах

## FAQ по админке

### Почему изменения не видны на другом устройстве?
Изменения хранятся локально. Для синхронизации:
1. Экспортируйте "Весь сайт" → замените `content.json` → git push
2. На другом устройстве обновите страницу (Ctrl+F5)

### Как восстановить данные?
Используйте кнопку "Сброс к исходному" или импортируйте резервный JSON.

### Курсы не отображаются?
Проверьте, что у курсов заполнены обязательные поля (название, школа, URL).

### Изображения не загружаются?
1. Проверьте URL изображения
2. Используйте HTTPS ссылки
3. Убедитесь, что хост разрешает hotlinking
